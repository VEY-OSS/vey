# Border Classic Ansible Role

The `border-classic` role configures the `vey-proxy` for a classic border deployment scenario.

This role utilizes the base `vey-proxy` role and provides specific configurations and tasks needed for the border classic profile.

## Tasks & Tags

*   `deploy`: Deploys the configuration and proxy binary.
*   `query-version`: Checks the currently running proxy version.
*   `clean-config`: Removes old configuration files.
*   `upgrade`: Upgrades the proxy binary.
*   `restart`: Restarts the proxy service.
*   `stop`: Stops the proxy service.
*   `config-all`: Generates all configurations and reloads the service.
*   `config-resolver`: Updates resolver configurations specifically and reloads.
*   `config-user`: Updates user configurations specifically and reloads.

## Variables

The following variables control the configuration generated by this role:

*   `border_classic_networks`: A dictionary defining the network configurations for the classic border proxy. Each network can define its own `dns_server`, `users` list, and other specific proxy settings.
*   `border_default_dns_server`: The default DNS server configuration used if a network-specific one is not provided.
*   `border_global_users`: An optional list of dictionaries representing a global pool of users. Each user dictionary must have a `name` key (e.g., `- name: "alice"\n  token: "xcrypt_hash_here"`).
*   `daemon_group`: Defines the instance name (set to `border-classic` in the playbook).

### Network Configuration Variables

Within each network in `border_classic_networks` (e.g. `border_classic_networks.my_net`), you can configure the following keys:
*   `port`: The HTTP proxy listening port.
*   `socks_port`: The SOCKS proxy listening port.
*   `users`: A list of strings (referencing `border_global_users`) or inline user dictionaries to authenticate against.
*   `listen_instance`: Number of instances to listen on (for load balancing).
*   `server_id`: A specific server ID.
*   `rsp_header_recv_timeout`: Timeout for receiving response headers.
*   `tls_client`: Configuration block for TLS client settings.
*   `dns_server`: DNS server mapping (overrides `border_default_dns_server`).
*   `resolve_strategy`: DNS resolution strategy (e.g., IPv4/IPv6 preference).
*   `resolve_redirection`: IP redirection rules.
*   `egress_network_filter`: Rules for filtering egress network destinations.
*   `socks_enable_udp_associate`: Boolean to enable UDP associate for SOCKS proxy.

### Configuring Users

In `border-classic`, users are mapped to the `static_users` configuration in `vey-proxy`. You can configure them in two ways:
1.  **Global Reference**: Define users in `border_global_users`. Then, in `border_classic_networks.<network>.users`, provide a list of strings matching the names of the global users.
2.  **Inline Definition**: In `border_classic_networks.<network>.users`, provide a list of dictionaries, where each dictionary directly defines a static user (e.g., `name` and `token`).

## Usage Example

```bash
# Apply all configurations
ansible-playbook -i inventory border-classic.yml -t config-all

# Only update user configurations
ansible-playbook -i inventory border-classic.yml -t config-user
```
