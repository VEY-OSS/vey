# Border Concise Ansible Role

The `border-concise` role configures the `vey-proxy` for a concise border deployment scenario, which generally involves a simplified or specific subset of configurations compared to the classic setup.

This role utilizes the base `vey-proxy` role.

## Tasks & Tags

Similar to `border-classic`, this role provides tasks for managing configurations and the daemon lifecycle:

*   `deploy`
*   `query-version`
*   `clean-config`
*   `upgrade`
*   `restart`
*   `stop`
*   `config-all`
*   `config-resolver`
*   `config-user`

## Variables

The following variables control the configuration generated by this role:

*   `border_concise_conf`: A dictionary of configurations for the concise border.
*   `border_default_dns_server`: The default DNS server configuration used if not specified in the concise config.
*   `daemon_group`: Defines the instance name (set to `border-concise` in the playbook).

### Configuration Dictionary Variables

The `border_concise_conf` dictionary accepts the following keys:
*   `port`: The HTTP proxy listening port.
*   `socks_port`: The SOCKS proxy listening port.
*   `users`: A list of inline user dictionaries to authenticate against.
*   `listen_instance`: Number of instances to listen on (for load balancing).
*   `server_id`: A specific server ID.
*   `rsp_header_recv_timeout`: Timeout for receiving response headers.
*   `tls_client`: Configuration block for TLS client settings.
*   `dns_server`: DNS server mapping (overrides `border_default_dns_server`).
*   `resolve_strategy`: DNS resolution strategy (e.g., IPv4/IPv6 preference).
*   `resolve_redirection`: IP redirection rules.
*   `egress_network_filter`: Rules for filtering egress network destinations.
*   `socks_enable_udp_associate`: Boolean to enable UDP associate for SOCKS proxy.

### Configuring Users

You can specify users by adding a `users` list to the `border_concise_conf` dictionary. This list should contain dictionaries defining the static users directly (e.g., `name` and `token`).

## Usage Example

```bash
# Upgrade the proxy binary
ansible-playbook -i inventory border-concise.yml -t upgrade

# Restart the service
ansible-playbook -i inventory border-concise.yml -t restart
```
