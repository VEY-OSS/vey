# Border Stream Ansible Role

The `border-stream` role configures the `vey-proxy` for handling streaming traffic at the network border.

This role utilizes the base `vey-proxy` role and provides configurations specific to stream proxying.

## Tasks & Tags

Standard proxy management and configuration tags are available:

*   `deploy`
*   `query-version`
*   `clean-config`
*   `upgrade`
*   `restart`
*   `stop`
*   `config-all`
*   `config-resolver`

## Variables

The following variables control the configuration generated by this role:

*   `border_stream_networks`: A dictionary defining network configurations specific to stream proxying.
*   `border_default_dns_server`: The default DNS server configuration used if a network-specific one is not provided.
*   `daemon_group`: Defines the instance name (set to `border-stream` in the playbook).

### Network Configuration Variables

Within each network in `border_stream_networks` (e.g. `border_stream_networks.my_net`), you can configure the following keys:
*   `dns_server`: DNS server mapping (overrides `border_default_dns_server`).
*   `resolve_strategy`: DNS resolution strategy (e.g., IPv4/IPv6 preference).
*   `egress_network_filter`: Rules for filtering egress network destinations.
*   `tcp_servers`: A dictionary defining the TCP stream servers for this network. Each server key acts as an identifier, and its value is a configuration dictionary containing:
    *   `port`: The listening port.
    *   `listen_instance`: Number of instances to listen on.
    *   `upstream`: The `proxy_pass` upstream address.

Note that stream proxies generally deal directly with traffic forwarding and do not use HTTP/SOCKS authentication, hence no user configurations are applied here.

## Usage Example

```bash
# Push all configurations and reload
ansible-playbook -i inventory border-stream.yml -t config-all
```
