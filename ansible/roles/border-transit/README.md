# Border Transit Ansible Role

The `border-transit` role configures the `vey-proxy` for acting as a transit proxy at the network border.

This role utilizes the base `vey-proxy` role and includes configurations specific to transit proxy operations.

## Tasks & Tags

Standard proxy management and configuration tags are available:

*   `deploy`
*   `query-version`
*   `clean-config`
*   `upgrade`
*   `restart`
*   `stop`
*   `config-all`
*   `config-resolver`
*   `config-user`

## Variables

The following variables control the configuration generated by this role:

*   `border_transit_conf`: A dictionary containing configurations for the transit proxy.
*   `border_default_dns_server`: The default DNS server configuration.
*   `daemon_group`: Defines the instance name (set to `border-transit` in the playbook).

### Configuration Dictionary Variables

The `border_transit_conf` dictionary accepts the following keys:
*   `port`: The HTTP proxy listening port.
*   `tls_name`: The expected TLS server name (SNI).
*   `users`: A list of inline user dictionaries to authenticate against.
*   `listen_instance`: Number of instances to listen on (for load balancing).
*   `server_id`: A specific server ID.
*   `rsp_header_recv_timeout`: Timeout for receiving response headers.
*   `tls_client`: Configuration block for TLS client settings.
*   `resolve_strategy`: DNS resolution strategy (e.g., IPv4/IPv6 preference).
*   `resolve_redirection`: IP redirection rules.
*   `egress_network_filter`: Rules for filtering egress network destinations.

### Configuring Users

You can specify users by adding a `users` list to the `border_transit_conf` dictionary. This list should contain dictionaries defining the static users directly (e.g., `name` and `token`).

## Usage Example

```bash
# Apply transit configuration and restart
ansible-playbook -i inventory border-transit.yml -t config-all,restart
```
