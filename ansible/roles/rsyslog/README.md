# Rsyslog Ansible Role

The `rsyslog` role is responsible for configuring the system's `rsyslog` daemon to handle and route the log files generated by `vey-proxy`.

This ensures that proxy logs are properly collected, rotated, and optionally forwarded to centralized logging systems.

## Tasks & Tags

*   `upgrade`: Upgrades the `rsyslog` daemon and its associated plugins. This task is tagged with `never` by default, so it must be explicitly requested using `-t upgrade`.

## Variables

This role does not require any specific variables to be configured. It relies on the host's default logging environment.

## Usage Example

You must specify the `upgrade` tag to actually perform the operations when using the `rsyslog.yml` playbook:

```bash
ansible-playbook -i inventory rsyslog.yml -t upgrade
```
